# Lost Media Emulator — Audit System

End-to-end method for scoring every preset and effect in the app against real degraded-medium references and producing a prioritized fix list for the v2 quality track.

---

## Artifact map

| Artifact | Path | Purpose |
|---|---|---|
| Rubric | `docs/audit/rubric.md` | Per-axis scoring criteria (5 axes, 1–5 scale, severity definitions) |
| Reference manifest | `docs/audit/references/manifest.json` | Cleared reference corpus (metadata/URLs only — no asset bytes in repo) |
| Reference license policy | `docs/audit/references/README.md` | Commercial-redistribute rule, acquisition checklist, scoring gate |
| Coverage report | `docs/audit/coverage.md` | Which of the 91 presets have been scored; regenerated by `report.mjs` |
| Scorecards | `docs/audit/scorecards/<medium>.json` | One JSON array per medium; each element is a `Scorecard` object |
| Fix list | `docs/audit/FIX-LIST.md` | Prioritized defect table (severity → effort); consumed by Track Q and Track E |
| Inventory helper | `src/lib/audit/inventory.ts` | `ALL_PRESET_NAMES` — the canonical list of all 91 preset names |
| Schema + validators | `src/lib/audit/schema.ts` | `Scorecard` / `ReferenceEntry` types, `RUBRIC_CRITERIA`, `validateScorecard`, `validateReference`, `presetCoverage` |
| Coverage report tool | `tools/audit/report.mjs` | Reads scorecards + inventory; prints coverage stats |
| Contact-sheet snippet | `tools/audit/contact-sheet.snippet.js` | Devtools console snippet; renders presets to a labelled PNG for visual scoring |

## Scorecard shape

Scorecards live in `docs/audit/scorecards/<medium>.json` as a JSON array. Each element must satisfy the `Scorecard` type from `src/lib/audit/schema.ts`:

```ts
type Scorecard = {
  id: string;                          // preset name or effect function name
  kind: "preset" | "effect";
  medium: string;                      // audit family (see note below)
  referenceRefs: string[];             // ≥1 id from references/manifest.json
  scores: {
    physicalPlausibility: number;      // 1–5: does the artifact match real physics?
    parameterBehaviour:   number;      // 1–5: does the slider traverse real degradation states?
    artifactCorrectness:  number;      // 1–5: right shape/placement/colour, not a generic filter?
    defaults:             number;      // 1–5: factory default is period-true out of the box?
    eraFit:               number;      // 1–5: all co-occurring artifacts correct for the claimed era?
  };
  severity: "none" | "low" | "med" | "high";
  note: string;                        // human-readable rationale, reference citations, caveats
};
```

**On `medium`:** the field denotes the **audit family / reference medium the scorecard was scored against** (e.g. `"vhs"`), not a strict per-preset format taxonomy. Presets that share a physical signal path with a reference medium may be proxy-scored against that reference at lower confidence — Betamax and U-matic share the colour-under composite path and were scored against the VHS reference in the pilot. These cards keep `medium: "vhs"` and disclose the proxy in their `note` field. If the audit later needs strict per-format tagging, add an optional `subjectFormat` field rather than abusing `medium`.

## Tooling commands (verified working, no network required)

### Coverage report

```bash
node_modules/.bin/vite-node tools/audit/report.mjs
```

Reads `src/lib/audit/inventory.ts` (all 91 preset names) and all scorecard files; prints a coverage summary and the list of outstanding presets. Regenerate after adding new scorecards.

### Contact sheet (visual scoring aid)

1. Start the dev server: `npm run dev` (port 5176).
2. Open the running app in a browser and open DevTools → Console.
3. Paste `tools/audit/contact-sheet.snippet.js` into the console.
4. Edit the `FILTER` regex near the top of the snippet to match the medium's preset names (e.g. `/vhs|betamax|u-matic/i`).
5. The snippet renders each matching preset onto a labelled contact sheet and triggers a PNG download.

Alternative (used in the VHS pilot): drive the live renderer via the app's preview / devtools — render each preset, screenshot the result, and compare side-by-side against reference frames extracted with `ffmpeg`.

### Validate scorecards

Paste this into a throwaway script and run with `node_modules/.bin/vite-node`:

```ts
import { validateScorecard, validateReference } from "@/lib/audit/schema";
import { readFileSync } from "fs";

const cards = JSON.parse(readFileSync("docs/audit/scorecards/vhs.json", "utf8"));
let ok = 0;
for (const c of cards) {
  const errs = validateScorecard(c);
  if (errs.length) console.error("INVALID", c.id, errs);
  else ok++;
}
console.log(`${ok}/${cards.length} valid`);
```

Replace `vhs.json` with the medium you are validating. `validateReference` takes the same shape from `manifest.json` entries.

## Per-medium scoring loop

Scoring the full 91 presets is this loop repeated per medium, not a one-shot task.

1. **Add a cleared reference.** Source ≥1 real piece of footage/imagery from the target medium that is licensed for commercial redistribution (`redistribute: true` — Public Domain, CC0, or equivalent). Add a `ReferenceEntry` to `docs/audit/references/manifest.json` following the acquisition checklist in `docs/audit/references/README.md`. No scoring may begin until this step is complete.

2. **Render a contact sheet.** Start `npm run dev`, run the contact-sheet snippet with a `FILTER` regex for the medium's presets, download the PNG. This is the visual material you score against.

3. **Score each preset and its core effects.** For every preset in the medium family (plus any shared effect functions, e.g. `advancedDropouts`), write one `Scorecard` JSON object scoring all five `RUBRIC_CRITERIA` axes on the 1–5 rubric scale. Every card must supply ≥1 `referenceRef` pointing to a manifest entry of the **same medium** (or a disclosed, lower-confidence proxy). Do not score without a reference.

4. **Validate scorecards.** Run the validate snippet above. All cards must pass `validateScorecard` with zero errors before committing.

5. **Regenerate coverage.** Run `node_modules/.bin/vite-node tools/audit/report.mjs`. Confirm the newly scored presets are removed from the outstanding list and `coverage.md` is updated.

6. **Append defects to FIX-LIST.md.** For every card with `severity` of `low`, `med`, or `high`, add a row to `docs/audit/FIX-LIST.md` following the column schema: `Effect / Preset | Medium | Problem (vs reference) | Severity | Est. effort | Fix lane`. Keep the problem text concrete and reference-grounded. Maintain sort order: severity high → med → low, then effort low → med within each tier.

## Pilot summary (VHS family)

The first loop iteration scored the VHS / consumer-tape family: 11 presets + 3 core effects = 14 scorecards, covering 12.1% (11/91) of the preset inventory. Findings: one high-severity defect (`advancedDropouts` shape), one med (`advancedHeadSwitching` band), and four low-severity items. Details in `docs/audit/scorecards/vhs.json` and `docs/audit/FIX-LIST.md`.

Seven presets (Hi8, Video8, U-matic, Betacam SP, HDV, D-VHS, Night Vision) were deferred pending format-matched cleared references.
